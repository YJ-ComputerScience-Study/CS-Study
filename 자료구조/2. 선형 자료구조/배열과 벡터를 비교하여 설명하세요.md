# 배열과 벡터를 비교하여 설명하세요.

배열과 벡터는 둘 다 데이터를 저장하는 선형 자료구조입니다.


먼저 배열은, 동일한 유형의 데이터가 연속된 메모리 공간에 저장된 데이터 구조입니다.
- 고정된 크기를 가지고, 
- 연속된 메모리 공간을 사용하여 유연성이 낮습니다. 
- 요소들이 연속적으로 저장되므로 캐시 효율이 좋습니다. 
- 데이터를 삽입하거나 삭제할 때, 나머지 요소들을 재배치해야 하므로 O(n)의 시간이 소요됩니다.

벡터는 동적 배열로, 필요에 따라 크기를 조정할 수 있는 데이터 구조입니다.
- 메모리 할당이 필요할 때마다 더 큰 배열을 할당하고 기존 요소를 복사합니다.
- 배열과 같이 인덱스를 사용해 요소에 접근할 수 있습니다.
- 끝부분에 요소를 추가하거나 제거하는 것은 O(1), 나머지 경우의 추가와 삭제는 O(n)의 시간이 소요됩니다.

따라서 배열은 요소의 개수가 고정되고 빠른 임의 접근이 필요한 경우에 적합하고, 벡터는 요소의 개수를 정확히 알지 못하여 동적으로 크기를 조절해야 할 때 적합합니다.
