# 연결 리스트의 장단점을 배열과 비교하여 설명하세요.

연결 리스트는 데이터를 감싼 노트를 포인터로 연결해 공간적인 효율성을 극대화시킨 자료 구조입니다.
이는 동적으로 데이터를 관리하기에 여러 장점을 가집니다.

1. 연결 리스트는 유연성이 뛰어납니다. 배열과 달리 데이터의 크기를 미리 지정할 필요가 없으므로 런타임 중에도 데이터의 추가와 삭제가 가능합니다.
2. 연결 리스트는 효율적인 메모리 사용이 가능합니다. 배열처럼 메모리에서 연속적인 위치에 있을 필요가 없으며, 단지 포인터로 노드들을 서로 연결시켜 주기만 하면 됩니다.
3. 연결 리스트는 데이터의 삽입과 삭제가 편리합니다. 배열은 삽입이나 삭제가 발생할 시 데이터를 재배치해야 하는데, 연결 리스트는 특정 노드의 참조만 변경하면 되기 때문에 상대적으로 간단한 작업만을 수행합니다.

반면에, 단점들도 존재합니다.

1. 연결 리스트는 직접 접근(랜덤 액세스)가 불가합니다. 배열은 인덱스를 사용하여 아무 원소나 직접 접근할 수 있지만, 연결 리스트가 특정 요소에 접근하기 위해서는 처음 원소부터 해당 요소까지 순차적으로 이동해야 합니다. 따라서 데이터 검색이 비효율적입니다.
2. 연결 리스트의 각 노드는 데이터 필드와 링크 필드를 가지는데, 이 링크 필드로 인해 배열에서는 사용하지 않는 추가적인 메모리를 사용하게 됩니다. 따라서 배열에 비해 메모리 오버헤드가 크다고 볼 수 있습니다.
