# 스프링 MVC와 struts의 차이는 무엇인가요?

Spring과 Struts 모두 Java 웹 어플리케이션을 개발하는 데 사용되지만, 여러 가지 차이점이 있습니다.

1. Spring은 웹 뿐만 아니라 자바 어플리케이션에도 특화된 프레임워크이지만, Struts는 웹에 특화된 프레임워크입니다.
2. Spring MVC는 여러 개의 분리된 구성요소로 이루어져 있는 컴포넌트 기반의 프레임워크이지만 Struts는 액션 기반의 프레임워크입니다. 이때 이 액션 클래스가 컨트롤러의 역할을 합니다.
3. Spring은 경량 컨테이너로서 의존성 주입을 통해 코드의 응집력을 높이는 데 비해, Struts는 이런 세세한 설정을 개발자가 직접 설정 파일에서 지정해야 합니다.
4. Spring이 Struts보다 확장성, 유연성이 높습니다.
5. 어노테이션을 주로 사용하는 Spring과 달리 Struts에서는 액션 매핑, 뷰 등을 설정할 때 xml 파일을 사용합니다.
