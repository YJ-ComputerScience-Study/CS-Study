# Clustered Index와 Non-Clustered Index의 차이를 설명하세요.

클러스터링 인덱스와 비 클러스터링 인덱스의 차이는 리프노드가 실제 레코드를 군집화해서 저장하는가, 아니면 따로 저장하는가에 있습니다. 

**클러스터링 인덱스**는 인덱스와 실제 레코드를 분리하지 않고 함께 군집화하여 저장하기 때문에, 리프 노드가 곧 실제 레코드입니다. 이때 레코드(실제 노드)들이 인덱스 컬럼 값을 기준으로 정렬된 상태를 유지합니다.

이에 반해, **비 클러스터링 인덱스**는 인덱스와 분리된 실제 레코드를 포인터로 가리키는 방식을 사용하고, 실제 레코드는 따로 저장합니다. 이때 리프 노드는 인덱스 컬럼 값과 실제 레코드의 주솟값을 가집니다. 인덱스 페이지는 정렬된 상태이고, 실제 데이터 페이지는 정렬되지 않은 상태이기 때문에 클러스터링 인덱스보다 데이터의 삽입/수정/삭제가 상대적으로 빠릅니다.
