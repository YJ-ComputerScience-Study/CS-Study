# 트랜잭션 격리 수준에 따라 발생하는 현상에 대해 말해 주세요.

트랜잭션 격리 수준에 따라 발생하는 현상에는 Phantom Read, Non-repeatable Read, Dirty Read 현상이 있습니다.


1. Phantom Read
- 이전 쿼리에 없던 유령 데이터가 나타나는 현상입니다.
- 트랜잭션 1이 같은 데이터를 두 번 조회할 때, 2회의 조회 시점 사이에 트랜잭션 2에서 새로운 데이터 하나를 추가하여 첫 번째 조회에서는 2개의 데이터, 두 번째 조회에서는 3개의 데이터가 조회되는 것으로 예를 들 수 있습니다.

2. Non-repeatable Read
- 같은 키를 가진 데이터를 여러 번 읽을 때 그 결과가 다르게 나타나는 현상입니다.
- 첫 번째 조회와 두 번째 조회 사이에 다른 트랜잭션에서 값을 변경하거나 삭제한다면 결과가 다르게 나타날 수 있습니다.

3. Dirty Read
- 다른 트랜잭션이 수정한 후 아직 커밋되지 않은 데이터를 읽게 되는 현상입니다.
- 해당 데이터가 commit될지 rollback될지 알 수 없으므로 데이터 부정합이 발생할 가능성이 높아집니다.



---
> [!NOTE]
> **Phantom Read와 Non-repeatable Read의 차이는?**
>
> Phantom Read는 다른 행이 추가로 선택되어 조회 결과가 다른 것이고, Non-repeatable Read는 행의 값이 달라져 조회 결과가 다른 것입니다.
